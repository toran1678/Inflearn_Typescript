# 📘 Inflearn TypeScript Study

## 🤝 스터디 소개

인프런 이정환 강사님의 한 입 크기로 잘라먹는 타입스크립트 강의 기반으로 진행됩니다.
타입스크립트의 개념을 공부, 이해하고 정리하여 함께 리뷰를 진행합니다.

---

## 🚀 스터디 진행방식

- 매주 ahr요일 오후 11시 59분까지 스터디 레포지토리에 학습한 내용을 제출한다.
- 스터디원들은 다른 스터디원의 정리 내용에서 궁금한 점이나 보완하면 좋은 점 등을 리뷰로 남긴다.

## 📤 과제 제출 방법

1. 해당 Repository를 본인 계정으로 **Fork**한다.
2. 본인 계정의 저장소에서 브랜치를 생성한다. (예 : `leekyuhyun`)
3. `본인이름` 폴더 안에 해당 주차 내용을 작성한다. (예: `이규현/1장_기초.md`)
4. 작성이 완료되면 원본 저장소의 `main` 브랜치로 **Pull Request**를 보낸다.
5. 스터디장의 Merge 후에는 반드시 **Sync fork**를 눌러 자신의 저장소를 최신화한다.

## ⏰ 제출 마감 및 기준

- 제출 기한 마감은 **매주 목요일 오후 11시 59분까지**이며, 마감 기한을 지키지 않을 경우 그 주 과제는 제출하지 않은 것으로 한다.
  - 특별한 사유가 있을시 예외
- 과제를 제출하지 않았더라고 개인적으로 진행 후 Pull Request를 보낼 수 있으며, 리뷰가 진행되지 않을 수도 있다.

## 📝 커밋 메시지

다음과 같은 커밋 메시지로 푸쉬를 진행합니다.

- `Docs: {주제} 정리` (ex. `Docs : 1장 타입 시스템 정리`)
- `Feat: {주제} 실습` (ex. `Feat : 인터페이스 실습 코드 추가`)
- `Fix: {수정내용}` (ex. `Fix : 제네릭 오타 수정`)

## 📂 폴더 구조

- 스티더 저장소는 다음과 같은 폴더 구조로 쌓일 에정입니다.

```
root/
├── 📂 목차
│   ├── 📂 팀원/
│   │   ├── 📄 md파일
│   │   └── 📄 실습이 있다면 실습파일
└── 📄 README.md
```
